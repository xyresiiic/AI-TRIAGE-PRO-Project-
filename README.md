## рҹ“Ң Overview

**AI Triage Pro** is a multimodal AI-powered emergency dispatch system that combines:
- рҹҺҷпёҸ **Audio Intelligence** вҖ” OpenAI Whisper transcribes emergency calls and detects 500+ medical conditions via acoustic fingerprinting
- рҹ‘ҒпёҸ **Vision Analysis** вҖ” CLIP zero-shot classifier assesses injury severity from uploaded images with confidence scoring
- вҡ–пёҸ **Weighted Fusion Engine** вҖ” Trust-aware engine combines both signals, penalizes suspicious inputs, and outputs a triage priority score
- рҹҡЁ **Instant Dispatch** вҖ” Priority classification triggers immediate ambulance dispatch recommendations with a full audit trail

The system was designed to assist emergency responders in triage decisions вҖ” reducing response time and improving patient outcomes.

## рҹҺҜ Triage Scoring System

| Score | Severity | Action | Response Time |
|-------|----------|--------|--------------|
| рҹ”ҙ **1** | CRITICAL | Immediate life threat | Dispatch within 2 minutes |
| рҹ”ҙ **2** | URGENT | Serious condition | Priority dispatch within 10 minutes |
| рҹҹ  **3** | MODERATE | Needs medical attention | Non-emergency transport |
| рҹ”ө **4** | MINOR | Minor injury | Walk-in clinic appropriate |
| рҹҹў **5** | STABLE | No emergency | Routine GP visit recommended |

## рҹҡҖ Getting Started

### Prerequisites

- Python 3.9+
- `ffmpeg` installed and added to PATH
  - Windows: [Download here](https://ffmpeg.org/download.html) вҶ’ extract вҶ’ add `/bin` to PATH
  - Linux/Mac: `sudo apt install ffmpeg` or `brew install ffmpeg`

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/xyresiiic/AI-TRIAGE-PRO-Project-.git
cd AI-TRIAGE-PRO-Project-

# 2. Install dependencies
pip install -r requirement.txt

# 3. Run the server
python app.py
```

### Usage

```
Open your browser вҶ’ http://localhost:5000
```

1. **ANALYZE** вҖ” Enter patient details, upload an audio call (MP3/WAV/M4A) and/or an injury image (JPG/PNG/WEBP)
2. **RUN ANALYSIS** вҖ” Watch the live pipeline process both modalities in real-time
3. **DISPATCH** вҖ” Review the integrated triage score and dispatch recommendation
4. **DASHBOARD** вҖ” Track all cases, severity distribution, and patient history

### CLI Usage (Fusion Engine Only)

```bash
python fusion_engine.py audio.mp3 image.jpg
```

## рҹ”Ң API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/` | Serves the frontend UI |
| `POST` | `/api/analyze/audio` | Transcribe + score an audio file |
| `POST` | `/api/analyze/image` | Classify + score an injury image |
| `POST` | `/api/analyze/fuse` | Weighted fusion of both results |
| `GET` | `/api/health` | Health check вҖ” returns version info |

### Example Fusion Request

```json
POST /api/analyze/fuse
{
  "audio_result": {
    "audio_triage_score": 1,
    "security": "VOICE_AUTHENTIC",
    "transcript": "my friend is bleeding from the head",
    "detected_condition": "head injury"
  },
  "vision_result": {
    "image_triage_score": 2,
    "security": "VERIFICATION_WARNING: No Camera Metadata",
    "top_category": "a deep cut or wound"
  }
}
```

### Example Response

```json
{
  "final_score": 2,
  "severity": "URGENT",
  "is_critical": true,
  "action": "PRIORITY 1 вҖ” IMMEDIATE AMBULANCE DISPATCH",
  "weights": { "audio": 1.0, "vision": 0.6 },
  "trust_flags": {
    "audio_suspicious": false,
    "vision_suspicious": true,
    "penalty_applied": true
  }
}
```

## рҹӣЎпёҸ Security Features

- **Acoustic Fingerprinting** вҖ” Detects synthetic/TTS voices using spectral centroid variance analysis
- **EXIF Metadata Verification** вҖ” Flags images with missing camera metadata as possibly synthetic
- **CLIP Relevance Filter** вҖ” Rejects non-medical images before injury classification
- **Trust-Weighted Fusion** вҖ” Suspicious signals receive reduced weight (0.5вҖ“0.6Г—) in the final score
- **Suspicion Penalty** вҖ” Score shifted toward stable when all submitted modalities are flagged
