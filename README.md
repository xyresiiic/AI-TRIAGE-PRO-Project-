## рЯУМ Overview

**AI Triage Pro** is a multimodal AI-powered emergency dispatch system that combines:
- рЯОЩпЄП **Audio Intelligence** вАФ OpenAI Whisper transcribes emergency calls and detects 500+ medical conditions via acoustic fingerprinting
- рЯСБпЄП **Vision Analysis** вАФ CLIP zero-shot classifier assesses injury severity from uploaded images with confidence scoring
- вЪЦпЄП **Weighted Fusion Engine** вАФ Trust-aware engine combines both signals, penalizes suspicious inputs, and outputs a triage priority score
- рЯЪ® **Instant Dispatch** вАФ Priority classification triggers immediate ambulance dispatch recommendations with a full audit trail

The system was designed to assist emergency responders in triage decisions вАФ reducing response time and improving patient outcomes.

## рЯЪА Getting Started

### Prerequisites

- Python 3.9+
- `ffmpeg` installed and added to PATH
  - Windows: [Download here](https://ffmpeg.org/download.html) вЖТ extract вЖТ add `/bin` to PATH
  - Linux/Mac: `sudo apt install ffmpeg` or `brew install ffmpeg`

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/yourusername/ai-triage-pro.git
cd ai-triage-pro

# 2. Install dependencies
pip install -r requirement.txt

# 3. Run the server
python app.py
```

### Usage

```
Open your browser вЖТ http://localhost:5000
```

1. **ANALYZE** вАФ Enter patient details, upload an audio call (MP3/WAV/M4A) and/or an injury image (JPG/PNG/WEBP)
2. **RUN ANALYSIS** вАФ Watch the live pipeline process both modalities in real-time
3. **DISPATCH** вАФ Review the integrated triage score and dispatch recommendation
4. **DASHBOARD** вАФ Track all cases, severity distribution, and patient history

### CLI Usage (Fusion Engine Only)

```bash
python fusion_engine.py audio.mp3 image.jpg
```

## рЯЫ°пЄП Security Features

- **Acoustic Fingerprinting** вАФ Detects synthetic/TTS voices using spectral centroid variance analysis
- **EXIF Metadata Verification** вАФ Flags images with missing camera metadata as possibly synthetic
- **CLIP Relevance Filter** вАФ Rejects non-medical images before injury classification
- **Trust-Weighted Fusion** вАФ Suspicious signals receive reduced weight (0.5вАУ0.6√Ч) in the final score
- **Suspicion Penalty** вАФ Score shifted toward stable when all submitted modalities are flagged
